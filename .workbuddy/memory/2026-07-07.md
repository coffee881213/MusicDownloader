## Session 14: Fix yt-dlp "Requested format is not available" error

- Error: `ERROR: [youtube] HegSBovl24I: Requested format is not available`
- **Root cause**: `player_client: ["web", "web_safari", "android", "ios"]` in YouTube download ydl_opts
  - When `player_client` is explicitly set, yt-dlp only queries those specific clients
  - The `web` client (tried first) returns progressive formats (mixed video+audio) but NOT adaptive DASH formats (audio-only streams like 251, 140, 249, 139)
  - `bestaudio` selector only matches audio-only streams → fails when no audio-only formats are returned
  - Without `player_client`, yt-dlp's default client strategy queries multiple clients and merges results, providing access to ALL formats including audio-only streams
- **Testing verified**:
  - With `player_client: ["web", ...]`: `bestaudio` → FAILED, `best` → format 18 (mp4 360p mixed)
  - Without `player_client`: `bestaudio` → format 251 (webm opus 139k audio-only) ✅
  - With `player_client: ["ios", "android", "web"]`: `bestaudio` → still failed (format 18 mp4)
  - Conclusion: ANY explicit `player_client` setting limits format availability vs yt-dlp defaults
- **Fixes applied** (3 layers of resilience):
  1. **Removed `player_client`** from YouTube download ydl_opts — let yt-dlp use its optimized default client selection
  2. **Added `/best` fallback** to format strings: `"bestaudio[ext=flac]/bestaudio[acodec=flac]/bestaudio/best"` — `best` always matches something (even mixed streams)
  3. **Added `format_sort`**: `["ext:flac", "acodec:flac", "abr", "asr"]` — prefer FLAC when available, then higher bitrate
  4. **Added retry mechanism**: if first attempt fails with "format not available", retries with `format: "best"` (simplest, always works)
  5. Same fixes applied to Bandcamp download function (format + retry)
- **Verified**: `HegSBovl24I` now resolves to format 140 (m4a, 130k audio) → FFmpeg converts to FLAC

  3. Config `concurrent-searches`: 2 → 1 (fewer concurrent server requests)
  4. CLI `--max-retries`: 3 → 1
  5. Added 5-second inter-query delay (`time.sleep(5)` between query variants) to avoid rapid-fire logins
  6. Updated probe message to mention "登录频率限制" (login rate limiting)
